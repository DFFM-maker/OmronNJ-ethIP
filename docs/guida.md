Guida Operativa: Installazione Omron Custom EthIP CollectorQuesta guida descrive come configurare il componente custom Python sul DX1 (o sulla VM di test) per leggere la struttura ScadaInterface.1. Preparazione dell'Ambiente (VS Code)Assicurati che l'ambiente Python sulla tua VM sia pronto:Apri il terminale in VS Code.Installa la libreria necessaria:pip install aphyt
Patch Critica (Python 3.11+): Se la tua VM usa Python 3.11 o superiore, devi applicare la correzione manuale al file aphyt/cip/cip.py come descritto nel piano di implementazione (cambiare le virgolette esterne delle f-string da singole a doppie).2. Salvataggio dei FileCrea una cartella di progetto in VS Code e salva il file omron_synapse_collector.py (lo trovi nel blocco di codice a fianco).3. Registrazione su SpeeDBee SynapseUna volta che il codice è pronto:Accedi alla Web UI di SpeeDBee Synapse (http://172.16.239.250:8120).Vai su Setting Menu (icona ingranaggio) -> Custom Components.Dipendenze: Nella tab Dependencies (Python), installa aphyt. Se non hai accesso a internet dalla VM, dovrai caricare il file .whl corrispondente.Componente: Nella tab Python(.py), clicca su Add e seleziona il file omron_synapse_collector.py.Riavvia SpeeDBee quando richiesto dal sistema.4. Configurazione nel Flow EditorIn Edit Mode, trascina il componente Omron Advanced EthIP Collector dalla sezione Custom nel pannello.Clicca sull'icona ingranaggio del componente e inserisci i parametri JSON:{
  "PLC_IP": "172.16.224.111",
  "Polling_Interval_ms": 1000
}
Wiring: - Collega l'uscita (out_port1) del Collector all'ingresso del tuo MSSQL Scada Manager.Collega l'uscita del Manager all'ingresso (in_port1) del Collector (per gestire i comandi di scrittura).5. Verifica dei TagUsa il Data Monitor sulla porta di uscita del collector per verificare che i nomi brevi (es. IsRunning, RecipeACK) ricevano correttamente i valori dal PLC NJ senza essere troncati.
